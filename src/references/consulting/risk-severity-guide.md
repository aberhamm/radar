# Risk Severity Guide

## Severity levels

### Critical
**Definition**: Something is actively broken, insecure, or will cause data loss.
**Examples**:
- Server secrets exposed in client-side code
- Authentication bypass in API routes
- Data fetching that returns wrong content to end users
- Build that silently fails, deploying stale content

**Action**: Must be fixed before any other work. Flag immediately to client.

### High
**Definition**: Something will cause significant problems if not addressed. May not be broken now but will break under known conditions.
**Examples**:
- Core dependency 2+ major versions behind with known breaking changes
- Missing editing secret validation (allows unauthorized content injection)
- No error boundary for CMS components (one broken component crashes the page)
- Preview mode using cached/stale data (editors can't see their changes)

**Action**: Should be addressed in the next sprint. Include in risk section of deliverable.

### Medium
**Definition**: Suboptimal pattern that increases maintenance cost or technical debt. Not broken but will slow the team down.
**Examples**:
- Core dependency 1 major version behind
- Missing .env documentation
- No security headers configured
- Inconsistent data fetching patterns across pages
- No TypeScript strict mode

**Action**: Plan to address. Include in recommendations.

### Low
**Definition**: Minor improvement opportunity. Mostly about code quality and developer experience.
**Examples**:
- Unused dependencies in package.json
- Minor version gaps in non-core packages
- Missing loading states for async operations
- Console warnings in development

**Action**: Nice to fix but don't block on it.

### Info
**Definition**: Observation worth noting. Not a problem, just context.
**Examples**:
- "This project uses Pages Router" (neutral fact)
- "Component factory is auto-generated by scaffold" (standard pattern)
- "Monorepo with 3 workspaces" (structural observation)

**Action**: Include for context. No fix needed.

## Scoring categories

| Category score | Criteria |
|---------------|----------|
| Red | Any critical finding, OR 3+ high findings |
| Yellow | Any high finding, OR 3+ medium findings |
| Green | Only medium, low, or info findings |
